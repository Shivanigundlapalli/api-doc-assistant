### User
What are the rate limits?

### Assistant
## Summary
The API enforces rate limits based on your subscription tier to ensure system stability. Exceeding these limits results in a `429 Too Many Requests` error.

## Endpoint / Concept
Rate Limits

## Details
| Tier | Requests per Minute | Requests per Day |
| :--- | :--- | :--- |
| **Free** | 100 | 10,000 |
| **Pro** | 1,000 | 500,000 |
| **Enterprise** | Custom (per SLA) | Custom (per SLA) |

### Rate Limit Headers
Every API response includes these headers to track your usage:
* `X-RateLimit-Limit`: Maximum requests allowed in the current window.
* `X-RateLimit-Remaining`: Requests remaining in the current window.
* `X-RateLimit-Reset`: Reset time in UTC epoch seconds.

## Important Notes
* If you receive a `429 Too Many Requests` status code, you must wait until the time specified in the `X-RateLimit-Reset` header before making further requests.
